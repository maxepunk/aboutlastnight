## Understanding Narrative Threads

### What Threads Are

**From COMPREHENSIVE_KNOWLEDGE.md and ROLLUP_LOGIC_AND_GAME_SEMANTICS.md:**

**Definition:** Thematic arcs that organize story content and create player choice opportunities

**Function:**
- Connect elements across characters and timeline events
- Provide thematic coherence to discoveries
- Create player-driven story pursuit paths
- Enable meaningful choice clusters (players pursuing a thread together)

**Key Insight:** Threads are THEMATIC (what story is told), not MOTIVATIONAL (why character cares). Same thread content can be framed through different goal cluster lenses.

### Why Thread Assignment Matters

**For Players:**
- Threads create thematic discovery paths
- Choice moments cluster around thread content
- Players pursuing same thread can collaborate/compete
- Thematic coherence makes story comprehensible

**For Game Balance:**
- Thread distribution affects which stories are discoverable
- Token concentration in threads creates choice cluster opportunities
- Empty threads mean missed thematic exploration
- Balanced coverage supports diverse play styles

**For Design:**
- Thread assignment is quality control for thematic coherence
- Helps identify content gaps
- Supports intentional narrative structure
- Enables thread-based puzzle and reward design

## Thread Assignment Decision Framework

### Step 1: Identify Token's Thematic Content

**Ask: What story does this token tell?**

**Corporate intrigue, competition, secrets?** → Consider Funding & Espionage
**Personal relationship problems, marriage, divorce?** → Consider Marriage Troubles
**Drug dealing, memory tech, extraction?** → Consider Memory Drug
**Community, parties, social networks?** → Consider Underground Parties
**Technical innovation, AI ethics, research?** → Consider Advanced Technology

**Key Principle:** Content determines primary thread, not owner's goal cluster.

### Step 2: Check Character Connection to Thread

**Ask: Does this character have narrative connection to thread?**

**Review character database fields:**
- Overview: Does character's backstory connect to thread?
- Timeline events: Is character involved in events developing this thread?
- Existing elements: Does character own other elements in this thread?

**If YES:** Strong candidate for thread assignment
**If NO:** Reconsider thread fit OR expand character's thread involvement

### Step 3: Consider Thread Needs

**Ask: Does this thread need more tokens?**

**Marriage Troubles, Memory Drug, Underground Parties, Advanced Technology:**
- Currently 0 tokens each
- Desperately need token content
- Any appropriate token should be assigned

**Funding & Espionage:**
- Has 1 token already
- Can support more, but less urgent
- Ensure new tokens develop thread differently

**Principle:** Prioritize assigning tokens to underdeveloped threads when content fits.

### Step 4: Validate Thematic Fit

**Ask: Does this token genuinely develop the thread's themes?**

**Strong thematic fit:**
- Content directly advances thread narrative
- Token reveals new aspect of thread
- Clear connection to thread's core questions

**Weak thematic fit:**
- Token only tangentially mentions thread
- Content doesn't develop thread themes
- Forced connection for assignment sake

**If weak fit:** Don't assign thread. Better to leave unassigned than force incorrect thread.

### Step 5: Check Goal Cluster Balance

**Ask: Does this thread have representation from multiple clusters?**

**Good thread coverage:**
- Multiple goal clusters represented
- Different perspectives on same thread
- Enables players from different clusters to engage

**Poor thread coverage:**
- Single cluster dominates
- Other clusters excluded
- Limits who can discover/engage

**Ideal:** Each thread has content from 2-4 different goal clusters, showing multiple perspectives on theme.

## Thread Assignment Anti-Patterns

### Anti-Pattern 1: Forced Assignment

**Mistake:** Assigning thread because element needs one, not because content fits

**Example:**
- Content: Sarah's random memory unrelated to marriage
- Forced assignment: Marriage Troubles (because Sarah = Marriage Troubles)
- Problem: Content doesn't develop Marriage Troubles themes

**Fix:** Only assign thread if content genuinely develops thematic arc. Better unassigned than wrongly assigned.

### Anti-Pattern 2: Multiple Thread Assignment

**Mistake:** Assigning multiple threads to single token because content touches both

**Example:**
- Content: Derek's party drug dealing
- Temptation: Assign both Memory Drug AND Underground Parties
- Problem: Database doesn't support multiple thread assignments effectively

**Fix:** Choose PRIMARY thread token develops most. Create separate tokens if both threads need development.

### Anti-Pattern 3: Thread by Character Instead of Content

**Mistake:** Assuming all character's content belongs to same thread

**Example:**
- Derek = Underground Parties character
- All Derek tokens = Underground Parties thread
- Problem: Derek's drug dealing content is Memory Drug thread

**Fix:** Assign based on content themes, not character identity.

### Anti-Pattern 4: Ignoring Thread Gaps

**Mistake:** Creating more tokens for already-developed threads while ignoring gaps

**Example:**
- Funding & Espionage has 1 token
- Creating 5 more Funding & Espionage tokens
- Marriage Troubles still has 0 tokens

**Fix:** Prioritize underdeveloped threads. Check thread distribution before creating tokens.

### Anti-Pattern 5: Vague Thread Fit

**Mistake:** Assigning thread based on tangential mention rather than thematic development

**Example:**
- Content: Character mentions attending a party once
- Assignment: Underground Parties thread
- Problem: Content doesn't develop party community themes

**Fix:** Thread assignment requires substantial thematic development, not just keyword mention.
